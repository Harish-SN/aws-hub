This 2025 edition covers active AWS networking, routing, connectivity, and global acceleration services used for secure, scalable, and hybrid cloud architectures.

Amazon VPC (Virtual Private Cloud)

Type: Isolated Virtual Network
Use Cases: Custom IP ranges, subnets, routing, security

  • Configurable IPv4 / IPv6 CIDR blocks.
  • Public and private subnets.
  • Security Groups and Network ACLs.
  • Subnet-level route tables.

Subnets

Type: Network Segmentation
Use Cases: Public vs private workloads

  • Each subnet is scoped to a single Availability Zone.
  • High availability is achieved using multiple subnets across AZs.

Route Tables

Type: Traffic Routing

  • Local VPC routing by default.
  • Routes to IGW, NAT Gateway, TGW, VPN, Peering.

Internet Gateway (IGW)

Type: Internet Access

  • Enables inbound and outbound internet connectivity.
  • Attached at the VPC level.

NAT Gateway

Type: Outbound Internet for Private Subnets

  • Allows outbound-only internet access.
  • Managed, highly available, and scalable.

VPC Peering

Type: VPC-to-VPC Connectivity

  • Private connectivity over AWS backbone.
  • No transitive routing.

Transit Gateway

Type: Enterprise Network Hub

  • Hub-and-spoke architecture.
  • Supports transitive routing.
  • Integrates VPN and Direct Connect.

VPC Endpoints

Type: Private AWS Service Access

  • Gateway endpoints: S3, DynamoDB.
  • Interface endpoints: ENI-based access to most AWS services.

AWS PrivateLink

Type: Private Service Exposure

  • Expose services privately across VPCs and accounts.
  • No public internet exposure.

Amazon Route 53

Type: DNS & Global Routing

  • Latency, weighted, geolocation routing.
  • Health checks and failover.
  • DNSSEC support.

Amazon CloudFront

Type: Content Delivery Network (CDN)

  • Global edge caching.
  • Integrated with Shield and WAF.
  • Supports signed URLs and cookies.

AWS Global Accelerator

Type: Network Traffic Accelerator

  • Anycast static IP addresses.
  • Improves TCP and UDP performance.

Amazon API Gateway

Type: Managed API Layer

  • Supports REST APIs, HTTP APIs, and WebSockets.
  • Private APIs via VPC endpoints.

AWS Direct Connect

Type: Dedicated Hybrid Connectivity

  • Private fiber connection.
  • Consistent and predictable latency.

AWS Site-to-Site VPN

Type: IPSec VPN

  • Encrypted tunnels over internet.
  • BGP routing support.

Elastic Load Balancing (ELB)

  • ALB: Layer 7 HTTP/HTTPS routing.
  • NLB: Layer 4, ultra-high throughput.
  • GWLB: Third-party firewall appliances.

AWS Network Firewall

Type: Managed Network Security

  • Stateful and stateless filtering.
  • Suricata-compatible rules.
