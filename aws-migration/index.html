AWS Migration & Transfer Services

Covers AWS migration, transfer, replication, backup, and modernization services used to move workloads, applications, databases, and data into AWS.

AWS Database Migration Service (DMS)

Type: Database Migration & Replication
Use Cases: Low-downtime homogeneous & heterogeneous DB migration

Overview

DMS migrates databases to AWS with minimal downtime.

  • Supports Oracle, MySQL, PostgreSQL, SQL Server, MongoDB, etc.
  • Change Data Capture (CDC) for continuous replication.
  • Works with AWS SCT for schema conversion.

AWS Schema Conversion Tool (SCT)

Type: Schema & Code Transformation
Use Cases: Cross-engine database migrations

Overview

SCT converts database schemas and code when migrating to new engines.

  • Converts tables, indexes, views, functions, procedures.
  • Includes SCT Data Extraction Agent for large databases.

SMS was used for migrating on-prem VMs to AWS but is now deprecated. Use AWS Application Migration Service (MGN) instead.

AWS Application Migration Service (MGN)

Type: Lift & Shift (Rehost)
Use Cases: Migrate physical & virtual servers to EC2

Overview

MGN is AWS‚Äôs primary large-scale server migration service.

  • Continuous block-level replication.
  • Automated testing and cutover.
  • Scales to thousands of servers.

AWS Backup

Type: Backup & Restore (Migration Support)
Use Cases: Backup during migration, DR, rollback

Overview

AWS Backup provides centralized, policy-based backup management.

  • EBS, RDS, DynamoDB, EFS, FSx, EC2.
  • Cross-account & cross-region backups.
  • Used to protect workloads during migration.

AWS DataSync

Type: Online Data Transfer
Use Cases: Move files between on-prem and AWS storage

Overview

Automated, high-speed data transfer service.

  • Transfers data to S3, EFS, FSx.
  • Optimized and encrypted transfer.
  • Supports NFS and SMB.

AWS Snow Family

Models: Snowcone, Snowball Edge, Snowmobile
Use Cases: Offline and edge data migration

Overview

Used when network transfer is slow or impractical.

  • Snowcone: Small, portable edge device.
  • Snowball Edge: Up to 80 TB + compute.
  • Snowmobile: Up to 100 PB per truck.

AWS Transfer Family

Protocols: SFTP, FTPS, FTP
Use Cases: Managed file transfer into AWS

Overview

Fully managed file transfer service backed by AWS storage.

  • Integrates with S3 and EFS.
  • Supports IAM, custom IdPs, and custom domains.

AWS Migration Hub

Type: Migration Tracking Dashboard
Use Cases: Central migration visibility

Overview

Provides a single place to track application migrations.

  • Tracks DMS, MGN, DataSync migrations.
  • Application-centric migration view.

Migration Hub Strategy Recommendations

Type: Migration & Modernization Planning
Use Cases: Application portfolio assessment

Overview

Analyzes workloads and recommends migration strategies.

  • 7 Rs: Rehost, Replatform, Refactor, Repurchase, Relocate, Retain, Retire.
  • Uses static code & dependency analysis.
