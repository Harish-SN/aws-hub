AWS Application Integration Services

AWS Application Integration Services help you build event-driven, loosely coupled, and scalable distributed systems using queues, event buses, APIs, and workflow engines.

Amazon SQS (Simple Queue Service)

Type: Managed Message Queue
Models: Standard, FIFO
Use Cases: Decoupling, buffering, async processing

Overview

SQS enables reliable message queuing between distributed components.

  • Standard: high throughput, at-least-once delivery.
  • FIFO: exactly-once processing and ordering.
  • Dead-letter queues supported.
  • Fully serverless.

Amazon SNS (Simple Notification Service)

Type: Pub/Sub Messaging
Use Cases: Fan-out, notifications, event broadcasting

Overview

SNS delivers messages to multiple subscribers.

  • Supports SQS, Lambda, HTTP, Email, SMS.
  • Used heavily for event fan-out patterns.

Amazon EventBridge

Type: Event Bus & Routing
Use Cases: Event-driven architecture, SaaS integration

Overview

EventBridge routes events from AWS services, custom apps, and SaaS providers.

  • Event filtering, retries, and schemas.
  • Supports custom event buses.
  • Includes Scheduler and Pipes.

Amazon EventBridge Pipes

Type: Point-to-Point Integration
Use Cases: Source ‚Üí target wiring without code

Overview

Pipes connect event sources directly to targets.

  • Built-in filtering and transformations.
  • Supports SQS, Kinesis, DynamoDB Streams.
  • Reduces Lambda glue code.

AWS Step Functions

Type: Serverless Workflow Orchestration
Use Cases: Orchestration, approvals, long-running workflows

Overview

Coordinates distributed applications using state machines.

  • Standard workflows: durable, up to 1 year.
  • Express workflows: high-throughput, low latency.
  • Visual workflow builder.

Amazon MQ

Type: Managed Message Broker
Protocols: ActiveMQ, RabbitMQ
Use Cases: Legacy messaging migration

Overview

Managed brokers for enterprise and legacy applications.

  • Supports JMS, AMQP, STOMP, MQTT.
  • Runs on broker instances (not serverless).
  • Best for lift-and-shift messaging.

Amazon API Gateway

Type: API Management
Use Cases: REST, HTTP, WebSocket APIs

Overview

Creates and manages scalable APIs.

  • Throttling, caching, auth, monitoring.
  • Deep Lambda integration.

AWS AppSync

Type: GraphQL API Service
Use Cases: Realtime & offline-capable apps

Overview

Fully managed GraphQL with real-time subscriptions.

  • Integrates with DynamoDB, Lambda, OpenSearch.
  • WebSocket-based subscriptions.

AWS AppFabric

Type: SaaS Integration & Observability
Use Cases: Unified SaaS security & logs

Overview

Normalizes security and audit data from SaaS applications.

  • Works with Slack, Okta, Zoom, M365.
  • Feeds Security Lake, SIEM tools.
