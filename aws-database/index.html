AWS Databases

AWS offers multiple purpose-built database services optimized for different data models, workloads, latency needs, and consistency guarantees. Includes serverless modes, global scaling, and analytics integrations ‚Äî ideal for certifications, interviews, and architecture design.

Amazon DynamoDB

Type: NoSQL (Key-Value & Document)
Schema: Flexible (schema-less items)
Serverless: ‚úÖ Fully serverless
Scaling: Auto-scaling (On-Demand / Provisioned)
Use Cases: Low-latency apps, IoT, gaming, e-commerce carts

Overview

DynamoDB is a fully serverless, low-latency NoSQL database with automatic replication across multiple AZs and single-digit millisecond performance at any scale.

  • SSD-backed with automatic multi-AZ replication.
  • DynamoDB Streams for event-driven architectures.
  • DAX for microsecond read latency.
  • Zero-ETL integration with Amazon Redshift.
  • Global Tables for multi-region active-active access.

Limitations: 400 KB item size, no joins, indexing via GSIs/LSIs only.

Amazon DocumentDB (MongoDB-compatible)

Type: NoSQL (Document)
Schema: Flexible (JSON-like documents)
Serverless: ‚úÖ Yes (Serverless cluster mode)
Scaling: Auto-scaling compute & storage
Use Cases: Catalogs, content management, user profiles

Overview

DocumentDB is a managed document database compatible with MongoDB APIs. Serverless mode removes cluster sizing while AWS manages capacity behind the scenes.

  • Compatible with MongoDB drivers & aggregation pipelines.
  • Automatic storage scaling up to 128 TB.
  • Managed backups, patching, and replication.
  • Supports vector-style similarity workloads via integrations.

Note: Not identical to open-source MongoDB ‚Äî feature parity differs.

Amazon RDS

Type: Relational (SQL)
Schema: Fixed
Serverless: ‚ùå No (Aurora provides serverless option)
Scaling: Vertical scaling + read replicas
Use Cases: Traditional applications, ERP, CRM

Overview

RDS is a managed relational database service supporting MySQL, PostgreSQL, Oracle, SQL Server, and MariaDB.

  • Multi-AZ deployments for high availability.
  • Read replicas for read scaling.
  • Automated backups and patching.
  • Blue/Green deployments for safer upgrades.

Amazon Aurora

Type: Relational (MySQL & PostgreSQL compatible)
Schema: Fixed
Serverless: ‚úÖ Aurora Serverless v2
Scaling: Auto-scaling compute & storage (up to 128 TB)
Use Cases: SaaS, cloud-native enterprise apps

Overview

Aurora is a cloud-native relational database with distributed storage replicated across 3 AZs (6 copies) for durability and performance.

  • Millisecond-level compute scaling (Serverless v2).
  • Up to 15 read replicas.
  • Zero-ETL integration with Amazon Redshift.
  • Global Database for low-latency global reads.

Limitation: Only MySQL & PostgreSQL compatible engines.

Amazon Neptune

Type: Graph Database
Schema: Flexible graph model
Serverless: ‚úÖ Neptune Serverless
Scaling: Auto-scaling compute & replicas
Use Cases: Fraud detection, recommendations, knowledge graphs

Overview

Neptune is a managed graph database optimized for highly connected data using Gremlin and SPARQL query languages.

  • Fast graph traversals.
  • ACID-compliant graph transactions.
  • Supports up to 15 read replicas.

Amazon Keyspaces (for Apache Cassandra)

Type: Wide-Column NoSQL
Schema: Flexible
Serverless: ‚úÖ Fully serverless
Scaling: Auto-scaling throughput
Use Cases: IoT telemetry, time-series, distributed systems

Overview

Amazon Keyspaces is a serverless Cassandra-compatible database using CQL with pay-per-request pricing.

  • No cluster management required.
  • Multi-region replication.
  • Integrates with analytics and ML pipelines.

Amazon QLDB (Quantum Ledger Database)

Type: Ledger Database
Schema: Fixed
Serverless: ‚úÖ Fully managed
Use Cases: Financial records, audits, supply chain

Overview

QLDB provides an immutable, cryptographically verifiable ledger under a centralized authority.

  • Append-only immutable journal.
  • Cryptographic verification of history.
  • SQL-compatible PartiQL queries.

Note: ‚ÄúQuantum‚Äù refers to cryptographic integrity, not quantum computing.

Amazon Timestream

Type: Time-Series Database
Schema: Flexible
Serverless: ‚úÖ Fully serverless
Scaling: Automatic hot/cold tiering
Use Cases: Metrics, IoT telemetry, observability

Overview

Timestream is a purpose-built time-series database optimized for high-ingestion, time-based queries.

  • Automatic cost-based data tiering.
  • Optimized time-series query engine.
  • Integrates with Kinesis, IoT Core, Lambda, Grafana.
