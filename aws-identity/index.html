AWS Identity & Federation Services

  • External IdP support (Azure AD, Okta, Google Workspace).

AWS STS (Security Token Service)

Type: Temporary Credential Service
Serverless: âœ…
Use Cases: Federation, AssumeRole, cross-account access

Overview

STS issues short-lived AWS credentials.

  • AssumeRole for cross-account access.
  • AssumeRoleWithSAML / AssumeRoleWithWebIdentity.
  • Core building block for Cognito and Identity Center.

AWS Organizations

Type: Multi-account Governance
Serverless: âœ…
Use Cases: SCPs, account structure, consolidated billing

Overview

Organizations centrally manages and governs AWS accounts.

  • Service Control Policies (SCPs).
  • Consolidated billing and budgets.
  • Foundation for Control Tower and Identity Center.

AWS Directory Service

Type: Managed Active Directory
Serverless: N/A
Use Cases: Microsoft AD workloads

Overview

Provides managed Microsoft Active Directory in AWS.

  • AWS Managed Microsoft AD.
  • AD Connector for on-prem directories.
  • Used with WorkSpaces, RDS SQL Server, FSx.

AWS RAM (Resource Access Manager)

Type: Resource Sharing
Serverless: N/A
Use Cases: Share AWS resources across accounts

Overview

Securely shares AWS resources between accounts.

  • Share VPC subnets.
  • Share Transit Gateways.
  • Share Route 53 Resolver rules.

AWS Control Tower

Type: Landing Zone Automation
Serverless: N/A
Use Cases: Multi-account governance

Overview

Automates enterprise-grade AWS landing zones.

  • Automated account provisioning.
  • Guardrails using SCPs.
  • Built on Organizations and Identity Center.

IAM Roles Anywhere

Type: External Identity Access
Serverless: N/A
Use Cases: On-prem and external workloads

Overview

Provides AWS access using X.509 certificates instead of access keys.

  • Ideal for on-prem servers and CI/CD systems.
  • Uses temporary credentials via STS.

AWS Identity Store

Type: User & Group Directory
Serverless: N/A
Use Cases: IAM Identity Center identities

Overview

The internal user and group directory used by IAM Identity Center.

  • Stores workforce users and groups.
  • Supports SCIM sync from external IdPs.
