AWS Storage Services

This revision covers all major AWS storage services including object, block, file storage, hybrid storage, cold archival, and data transfer tools.

Amazon EBS (Elastic Block Store)

Type: Block Storage for EC2
Use Cases: Databases, low-latency transactional workloads

Overview

EBS provides persistent block storage volumes attached to EC2 instances.

  • Volume types: gp3, io2, io2 Block Express.
  • Volumes are single-AZ with snapshot-based durability.
  • Snapshots are stored in S3 and can be copied cross-region.

Amazon EFS (Elastic File System)

Type: Managed NFS File System
Use Cases: Shared storage for EC2, Lambda (VPC), containers

Overview

EFS is a fully managed, elastic file system that scales automatically.

  • POSIX-compliant file system.
  • Multi-AZ by default.
  • Storage classes: Standard, IA, One Zone.

Amazon FSx Family

Type: High-performance File Systems
Use Cases: HPC, ML, Windows workloads, enterprise file systems

Overview

FSx provides purpose-built, high-performance file systems.

  • FSx for Windows File Server
  • FSx for Lustre (HPC + S3 integration)
  • FSx for OpenZFS
  • FSx for NetApp ONTAP

Amazon S3 Glacier

Type: Archival Storage
Use Cases: Compliance, long-term retention

Overview

Low-cost archival storage with configurable retrieval times.

  • Glacier Instant Retrieval
  • Glacier Flexible Retrieval
  • Glacier Deep Archive

AWS Backup

Type: Centralized Backup Management
Use Cases: Backup automation and compliance

Overview

Central service to automate and manage backups across AWS.

  • Supports EBS, EFS, RDS, DynamoDB, FSx, S3.
  • Cross-account and cross-region support.

AWS Storage Gateway

Type: Hybrid Cloud Storage
Use Cases: On-premises to AWS integration

Overview

Connects on-premises environments to AWS storage.

  • File Gateway (NFS/SMB ‚Üí S3)
  • Volume Gateway (cached/stored volumes)
  • Tape Gateway (virtual tape library)

AWS Snow Family

Type: Offline & Edge Data Transfer
Use Cases: Large-scale migrations, edge computing

Overview

Physical devices for offline data transfer and edge workloads.

  • Snowcone
  • Snowball Edge
  • Snowmobile (up to 100 PB)

AWS DataSync

Type: Online Data Transfer
Use Cases: NFS/SMB migrations, replication

Overview

High-speed, automated data transfer service.

  • On-premises ‚Üî AWS transfers.
  • AWS storage service to service transfers.

AWS Transfer Family

Type: Managed File Transfer (SFTP/FTP/FTPS)
Use Cases: Secure file ingestion to S3/EFS

Overview

Fully managed file transfer endpoints.

  • Supports SFTP, FTP, FTPS.
  • Integrates with IAM, AD, Cognito.
